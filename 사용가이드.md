## 1. 시작하기 전에 — 앱 구조 이해

이 시스템은 **3단계 구조**로 이루어져 있습니다.

```
사업 (Project)
  └── 비목 (예산 항목)
        └── 지출내역 (개별 지출 기록)
```

| 용어 | 설명 | 예시 |
|------|------|------|
| **사업** | 예산을 관리할 프로젝트 단위 | 2025년 스마트시티 사업 |
| **비목** | 사업 예산을 나누는 항목 | 인건비, 운영비, 장비비, 회의비 |
| **지출내역** | 실제로 돈이 나간 건별 기록 | 2025.03.15 / AWS 서버비 / 운영비 / ₩450,000 |

**핵심 개념 — 집행률**

```
집행률 = 집행액 ÷ 예산 × 100
잔액   = 예산 - 집행액
```

집행률에 따라 색상으로 상태를 표시합니다.

| 색상 | 상태 | 집행률 범위 |
|------|------|------------|
| 🟢 초록 (양호) | 정상 범위 | 70% 미만 |
| 🟡 노랑 (주의) | 예산 소진 주의 | 70% ~ 90% |
| 🔴 빨강 (위험) | 예산 거의 소진 | 90% 이상 |

## 2. 프로그램 실행 방법

앱을 사용하려면 **백엔드**와 **프론트엔드** 서버를 모두 켜야 합니다.
터미널(명령 프롬프트) 창 **2개**를 열어주세요.

### 터미널 1번 — 백엔드 서버

```
cd C:\Users\junyo\Desktop\사업예산 관리\backend
.venv\Scripts\activate
uvicorn app.main:app --reload --port 8000
```

아래와 같이 나오면 정상입니다.
```
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### 터미널 2번 — 프론트엔드 서버

```
cd C:\Users\junyo\Desktop\사업예산 관리\frontend
npm run dev
```

아래와 같이 나오면 정상입니다.
```
  VITE v5.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

### 브라우저 접속

두 서버가 모두 켜진 상태에서 브라우저를 열고 아래 주소로 접속합니다.

```
http://localhost:5173
```

> **주의**: 앱을 사용하는 동안 터미널 창 2개를 닫으면 안 됩니다.

## 3. 첫 사용 시 입력 순서 (필독)

처음 사용할 때는 **반드시 아래 순서**대로 입력해야 합니다.
순서를 지키지 않으면 일부 기능이 작동하지 않습니다.

```
STEP 1  사업 등록        ← 가장 먼저! 사업명과 전체 예산 입력
   ↓
STEP 2  비목 설정        ← 예산을 항목별로 나누기 (인건비, 운영비 등)
   ↓
STEP 3  지출 입력        ← 실제 지출이 발생할 때마다 기록
   ↓
STEP 4  현황 확인        ← 대시보드, 집행현황, 월별 차트 조회
```

> **왜 이 순서인가요?**
> 지출을 입력할 때 어떤 '비목'에 해당하는지 선택해야 합니다.
> 비목이 없으면 지출을 입력할 수 없고, 사업이 없으면 비목을 만들 수 없습니다.

## 4. 사업 등록

### 4-1. 사업 생성 진입

화면 왼쪽 사이드바에서 **"+ 새 프로젝트 추가"** 를 클릭하거나,
대시보드에서 **"새 프로젝트"** 버튼을 클릭합니다.

### 4-2. 입력 항목

| 항목 | 필수 여부 | 설명 | 입력 예시 |
|------|----------|------|----------|
| **사업명** | 필수 | 사업을 구분하는 이름 | 2025년 스마트시티 사업 |
| **사업 설명** | 선택 | 사업에 대한 간단한 설명 | 시내버스 노선 최적화 연구 |
| **시작일** | 선택 | 사업 시작 날짜 | 2025-01-01 |
| **종료일** | 선택 | 사업 종료 날짜 | 2025-12-31 |
| **전체 예산** | 필수 | 사업 전체 예산 (원 단위) | 150,000,000 |

### 4-3. 주의사항

- 사업은 **최대 3개**까지만 등록할 수 있습니다.
- 예산 입력 시 쉼표 없이 숫자만 입력합니다. (예: `150000000`)
  입력하면 자동으로 `₩150,000,000` 형식으로 표시됩니다.
- 사업을 생성한 후에도 언제든지 수정할 수 있습니다.

### 4-4. 저장 후

**"사업 생성"** 버튼을 누르면 해당 사업의 집행현황 페이지로 이동합니다.
이제 바로 **STEP 2: 비목 설정**으로 넘어가세요.

## 5. 비목 설정

비목이란 전체 예산을 용도별로 나눈 항목입니다.
예를 들어 전체 예산 1억 원을 인건비 5천만, 운영비 3천만, 장비비 2천만으로 나누는 것입니다.

### 5-1. 비목 관리 진입

사업 집행현황 페이지 우측 상단의 **"비목 관리"** 버튼을 클릭합니다.

또는 왼쪽 사이드바에서 해당 사업 → **"비목 관리"** 를 클릭합니다.

### 5-2. 비목 추가

1. **"+ 비목 추가"** 버튼 클릭
2. 비목명 입력 (예: `인건비`)
3. 배정 예산 입력 (예: `50000000`)
4. 체크 버튼(✓) 클릭하여 저장

### 5-3. 비목 구성 예시

| 비목명 | 배정 예산 | 사용 예 |
|--------|----------|--------|
| 인건비 | 80,000,000 | 직원 급여, 외주 인건비 |
| 운영비 | 25,000,000 | 사무용품, 인쇄비, 소프트웨어 |
| 장비비 | 30,000,000 | 서버, 노트북, 기자재 구입 |
| 회의비 | 10,000,000 | 회의실 대관, 다과비, 출장비 |
| 기타 | 5,000,000 | 그 외 잡비 |

### 5-4. 비목 설정 시 주의사항

**예산 배분 합계 확인**
비목별 배정 예산 합계가 전체 예산을 초과하면 **경고 메시지**가 표시됩니다.
초과해도 저장은 가능하지만, 실제 예산 관리에 오류가 생길 수 있으니 맞춰주세요.

```
전체 예산:          ₩150,000,000
비목 배정 합계:     ₩150,000,000   ← 일치해야 정확한 관리 가능
미배정:                     ₩0
```

**비목 수정/삭제**
- 비목명 오른쪽 연필(✏) 아이콘: 이름 또는 배정 예산 수정
- 휴지통(🗑) 아이콘: 비목 삭제
- **주의**: 비목을 삭제하면 해당 비목의 지출내역도 함께 삭제됩니다.

## 6. 지출 입력

실제로 돈을 지출했을 때 기록합니다.
카드 결제, 계좌이체 등 지출이 발생할 때마다 입력합니다.

### 6-1. 지출 입력 진입

다음 중 하나로 진입합니다.

- 사업 집행현황 페이지 → **"+ 지출 입력"** 버튼 (주황색)
- 지출내역 페이지 → **"+ 지출 입력"** 버튼
- 왼쪽 사이드바 → 해당 사업 → **"지출내역"** → **"+ 지출 입력"**

### 6-2. 입력 항목 설명

| 항목 | 필수 여부 | 설명 | 입력 예시 |
|------|----------|------|----------|
| **지출일** | 필수 | 실제 결제/지출된 날짜 | 2025-03-15 |
| **비목** | 필수 | 해당 지출의 예산 항목 | 운영비 |
| **내용(적요)** | 필수 | 무엇을 샀는지 간단히 | AWS 서버 이용료 3월분 |
| **지출처** | 선택 | 어디서 결제했는지 | Amazon Web Services |
| **사용카드번호** | 선택 | 사용한 카드 번호 | 1234-5678-9012-3456 |
| **금액** | 필수 | 지출 금액 (원 단위) | 450000 |

### 6-3. 카드번호 입력 방법

카드번호를 입력하면 **자동으로 마스킹** 처리됩니다.
실제 카드번호 16자리를 입력하면 `****-****-****-3456` 형태로 저장됩니다.

```
입력:  1234-5678-9012-3456
저장:  ****-****-****-3456   (뒤 4자리만 표시)
```

### 6-4. 금액 입력 방법

금액 칸은 숫자만 입력합니다. 쉼표 없이 입력해도 자동으로 표시됩니다.

```
입력:  450000
표시:  ₩450,000
```

### 6-5. 저장 후

**"저장"** 버튼을 누르면 지출내역 목록으로 이동합니다.
방금 입력한 지출이 목록 상단에 표시됩니다.

## 7. 대시보드 — 전체 현황 확인

왼쪽 사이드바 최상단 **"대시보드"** 를 클릭합니다.

### 대시보드에서 볼 수 있는 정보

**프로젝트 요약 카드**

등록된 사업이 카드 형태로 표시됩니다.

```
┌─────────────────────────────┐
│ 2025년          65%         │
│ 스마트시티 사업              │
│ ████████████░░░░ 65%        │
│ 집행액 ₩97,500,000          │
│ 잔액   ₩52,500,000          │
│ 전체 예산 ₩150,000,000  →   │
└─────────────────────────────┘
```

카드를 클릭하면 해당 사업의 상세 집행현황으로 이동합니다.

**전체 집행현황**

모든 사업을 합산한 전체 예산 집행 현황을 가로 진행 바로 표시합니다.

**최근 지출 내역**

가장 최근에 입력된 지출 6건을 표시합니다.

## 8. 사업 상세 — 집행현황 확인

왼쪽 사이드바에서 사업명을 클릭 → **"집행현황"** 을 선택합니다.

### 상단 수치 카드

| 카드 | 설명 |
|------|------|
| **전체예산** | 사업 등록 시 설정한 총 예산 |
| **집행액** | 지금까지 입력된 지출 합계 |
| **잔액** | 전체예산 - 집행액 |
| **집행률** | 도넛 차트로 % 표시 |

### 비목별 집행현황 표

| 열 | 설명 |
|----|------|
| 비목명 | 예산 항목 이름 |
| 배정예산 | 비목에 설정한 예산 |
| 집행액 | 해당 비목에서 지출된 합계 |
| 잔액 | 배정예산 - 집행액 |
| 집행률 | 진행 바로 시각적 표시 |
| 상태 | 양호/주의/위험 뱃지 |

잔액이 **음수(빨간색)**이면 해당 비목의 예산을 초과 집행한 것입니다.

## 9. 지출내역 조회 및 수정

왼쪽 사이드바 → 사업 → **"지출내역"** 클릭

### 필터 기능

지출내역이 많아지면 필터를 활용해 원하는 내역만 조회합니다.

| 필터 | 사용법 |
|------|--------|
| **비목** | 드롭다운에서 특정 비목 선택 |
| **시작일 ~ 종료일** | 날짜 범위 지정 |

필터 설정 후 **"조회"** 버튼을 클릭합니다.
**"초기화"** 버튼을 누르면 전체 내역이 다시 표시됩니다.

### 지출 수정

수정하려는 지출 행의 **연필 아이콘(✏)** 을 클릭합니다.
지출 입력 폼이 열리며 내용을 수정한 후 **"저장"** 을 클릭합니다.

### 지출 삭제

삭제하려는 지출 행의 **휴지통 아이콘(🗑)** 을 클릭합니다.
**"정말 삭제하시겠습니까?"** 확인창이 뜨면 **"삭제"** 를 클릭합니다.

> 삭제된 지출은 복구할 수 없습니다. 신중하게 확인 후 삭제하세요.

## 10. 월별 집행현황 및 차트

왼쪽 사이드바 → 사업 → **"월별 현황"** 클릭

### 월별 집행현황 에어리어 차트

1월부터 12월까지 월별 집행액 추이를 면적 차트로 표시합니다.
차트 위에 마우스를 올리면 해당 월의 집행액이 팝업으로 표시됩니다.

### 월별 집행 통계 표

| 열 | 설명 |
|----|------|
| 월 | 1월 ~ 12월 |
| 비목별 금액 | 각 비목에서 해당 월에 집행된 금액 |
| 합계 | 해당 월 전체 집행액 (주황색 강조) |
| 마지막 행 합계 | 연간 비목별 총 집행액 |

**연도 변경**: 상단 드롭다운에서 연도를 선택합니다.

### 비목별 월별 추이 라인 차트

각 비목이 다른 색의 선으로 표시되어 비목별 지출 패턴을 비교할 수 있습니다.

## 11. 엑셀 다운로드

### 다운로드 방법

다음 두 곳에서 다운로드 버튼을 찾을 수 있습니다.

- **사업 집행현황 페이지** → 우측 상단 **"엑셀 다운로드"** 버튼
- **월별 집행현황 페이지** → 우측 상단 **"엑셀 다운로드"** 버튼

버튼을 클릭하면 자동으로 파일이 다운로드됩니다.

### 파일 구성

다운로드된 `.xlsx` 파일은 **시트 2개**로 구성됩니다.

**시트 1: 지출내역**

| 번호 | 날짜 | 비목 | 내용(적요) | 지출처 | 카드번호 | 금액 |
|------|------|------|-----------|--------|---------|------|

## 12. 자주 묻는 질문 (FAQ)

**Q. 사업을 4개 이상 만들려고 했는데 오류가 납니다.**
A. 이 시스템은 최대 3개의 사업만 관리할 수 있습니다. 완료된 사업을 삭제하면 새 사업을 등록할 수 있습니다.

---

**Q. 지출을 입력했는데 집행률이 변하지 않습니다.**
A. 페이지를 새로고침(F5)하거나 다른 메뉴로 갔다가 돌아오면 반영됩니다.

---

**Q. 비목을 만들지 않고 지출을 입력하려고 했는데 비목 목록이 비어있습니다.**
A. 지출 입력 전에 반드시 비목을 먼저 등록해야 합니다. 비목 관리 페이지로 이동해 비목을 추가하세요.

---

**Q. 카드번호를 잘못 입력했습니다. 수정할 수 있나요?**
A. 지출내역 목록에서 해당 지출의 수정 버튼을 클릭하면 카드번호를 포함한 모든 항목을 수정할 수 있습니다.

---

**Q. 비목 예산을 초과해서 지출을 입력했습니다. 문제가 되나요?**
A. 시스템에서는 막지 않습니다. 단, 집행현황 표에서 잔액이 빨간색 음수로 표시되어 초과 집행을 알 수 있습니다.

---

**Q. 지출내역이 많아서 찾기 어렵습니다.**
A. 지출내역 페이지의 필터 기능을 사용하세요. 비목별 또는 날짜 범위로 검색할 수 있습니다.

---

**Q. 앱을 껐다가 다시 켜도 데이터가 남아있나요?**
A. 네, 모든 데이터는 데이터베이스 파일(`backend/budget.db`)에 저장되므로 프로그램을 종료해도 데이터는 유지됩니다.

---

**Q. 서버를 실행했는데 브라우저에서 화면이 안 뜹니다.**
A. 다음을 확인해 주세요.
1. 터미널 1 (백엔드)와 터미널 2 (프론트엔드) 모두 실행 중인지 확인
2. 백엔드가 8000번 포트, 프론트엔드가 5173번 포트인지 확인
3. 브라우저 주소창에 `http://localhost:5173` 정확히 입력
